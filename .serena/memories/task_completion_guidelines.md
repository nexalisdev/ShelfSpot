# Task Completion Guidelines

## Backend
1. **Write Code**: Follow the module-based architecture. Use DTOs for data validation.
2. **Run Tests**: Ensure all tests pass using `yarn test`.
3. **Run Linting**: Check for linting errors using `yarn lint`.
4. **Run TypeScript Check**: Ensure there are no type errors using `yarn tsc`.
5. **Update Documentation**: Update relevant documentation in `install/documentation/`.
6. **Commit Changes**: Use conventional commit messages.

## Frontend
1. **Write Code**: Follow Next.js conventions. Use modular components.
2. **Run Tests**: Ensure all tests pass (if applicable).
3. **Run Linting**: Check for linting errors using `yarn lint`.
4. **Run TypeScript Check**: Ensure there are no type errors using `yarn tsc`.
5. **Update Documentation**: Update relevant documentation in `install/documentation/`.
6. **Commit Changes**: Use conventional commit messages.

## General
- **Docker**: Ensure Docker images build successfully and containers run without issues.
- **Pull Requests**: Ensure all checks pass before creating a pull request.
