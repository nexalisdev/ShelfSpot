# Code Style and Conventions

## Backend
- **Language**: TypeScript
- **Framework**: NestJS
- **Linting**: ESLint configured with `eslint.config.mjs`
- **Formatting**: Prettier
- **Architecture**: Module-based architecture with DTOs for data validation

## Frontend
- **Language**: TypeScript
- **Framework**: Next.js
- **Styling**: TailwindCSS
- **Linting**: ESLint configured with `eslint.config.mjs`
- **Formatting**: Prettier

## General
- **Commit Messages**: Follow conventional commit messages.
- **File Naming**: Use kebab-case for files and directories.
- **TypeScript**: Use strict typing and interfaces wherever possible.
